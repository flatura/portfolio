# MyLovelyGarden (MLG): SaaS-платформа для ботанических организаций

**Статус:** MVP / подготовка к пилоту  
**Роль:** Co-founder, System Designer, Backend-oriented Engineer  
**Платформа:** Web

## О проекте

**MyLovelyGarden** — прикладная SaaS-платформа для ботанических садов, питомников, селекционеров и частных коллекций растений.

Проект возник как ответ на типовую проблему предметной области: учет растений, наблюдений, происхождения, размещения и связанных операций часто ведется в разрозненных таблицах, локальных базах или устаревших специализированных решениях. Это затрудняет поиск, обмен данными между организациями, единообразное ведение справочной информации и работу с географически привязанными объектами.

Цель проекта — создать единую систему для ведения ботанических коллекций и связанных процессов с опорой на:
- централизованный справочный контур таксонов;
- ролевую модель доступа с учетом организационного контекста;
- поддержку пространственных данных и картографических сценариев;
- трассируемость жизненного цикла экземпляров растений.

## Ключевые системные решения

### 1. Разделение frontend и backend
Проект развивается как разделенная web-архитектура с отдельным frontend-клиентом и backend API. Это упрощает развитие интерфейса, делает систему менее связанной и создает основу для дальнейшего расширения клиентских каналов.

### 2. Soft multi-tenancy
Для изоляции данных выбрана модель soft multi-tenancy в рамках общей схемы хранения. Это позволяет снизить стоимость владения на раннем этапе, упростить сопровождение и сохранить возможность общей аналитики.

### 3. Контекстно-зависимая ролевая модель
Модель доступа учитывает не только глобальную роль пользователя в системе, но и его принадлежность к конкретной организации и контексту работы. Такой подход лучше соответствует B2B-сценариям, где один и тот же пользователь может иметь разные права в разных организациях.

### 4. Поддержка пространственных данных
Система проектируется с учетом работы с географически привязанными объектами: размещением растений, участками, полигонами, теплицами и другими пространственными сущностями. Пространственный контур рассматривается как полноценная часть предметной модели, а не как вторичная визуальная надстройка.

### 5. Гибридная модель состояния и истории
Для учета жизненного цикла экземпляров растений используется гибридный подход: актуальное состояние хранится отдельно от истории событий и изменений. Это позволяет одновременно поддерживать удобное операционное чтение текущих данных и сохранять трассируемость действий, перемещений и изменений состояния.

### 6. Многоуровневая модель доверия в рамках обмена данными
Система имеет гибко гранулированное разделение доступа к спискам: администратор может выдать доступ к той или иной коллекции всему миру, либо пользователям только верифицированных организаций, либо отдельновзятым пользователям (адресный шеринг). Права на доступ проверяются бизнес-логикой для каждого запроса.

## Инфраструктурный подход

На этапе MVP проект ориентирован на практичное развертывание с контролируемой операционной сложностью. Архитектура разворачивания строится так, чтобы не усложнять старт, но сохранять возможность дальнейшего переноса в более зрелую облачную среду.

Текущий подход включает:
- контейнеризацию сервисов;
- раздельную сборку клиентской и серверной частей;
- внешнее объектное хранилище для мультимедийных данных;
- базовую CI/CD-автоматизацию;
- сохранение технологической и архитектурной возможности для последующего масштабирования.
