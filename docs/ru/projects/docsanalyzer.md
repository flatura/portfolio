## Confluence Documentation Practice Analyzer

**Тип:** Internal tooling / Analytics chapter initiative  
**Роль:** System Analyst, Internal Tooling Developer  
**Статус:** implemented ad-hoc utility

### Контекст

В рамках активности в ядре чаптера аналитики появилась потребность лучше понимать, как аналитическая документация создаётся и поддерживается в Confluence: объём материалов, авторство, структура, использование согласованных шаблонов оформления и соответствие внутренним документационным практикам.

Целью было не “оценивать людей”, а сделать документационные практики более наблюдаемыми и создать основу для повышения консистентности, онбординга и переиспользования знаний внутри аналитических команд.

### Проблема

В Confluence находилось большое количество аналитических материалов, созданных разными авторами и командами. Ручная проверка была медленной и непоследовательной. Было сложно быстро ответить на практические вопросы:

- какие материалы создал или обновлял конкретный аналитик;
- используются ли в документах согласованные корпоративные макросы и шаблоны оформления;
- соответствуют ли страницы принятым документационным соглашениям;
- где документационные практики сильны, а где требуют выравнивания;
- какие страницы можно использовать как хорошие референсные примеры для чаптера аналитики.

### Решение

Была разработана ad-hoc консольная утилита, которая через Confluence API выгружала страницы, созданные или обновлённые заданным аналитиком по доменному логину, и подготавливала данные для дальнейшего анализа.

Утилита позволяла извлекать метаданные страниц и анализировать содержимое Confluence storage format, включая наличие выбранных макросов, используемых во внутренних стандартах документации.

### Зона ответственности

- Получение страниц Confluence по автору / доменному логину.
- Извлечение метаданных: заголовок, space, URL, автор, дата создания, дата обновления.
- Анализ storage-format содержимого страниц.
- Детекция выбранных Confluence-макросов.
- Подготовка структурированной выгрузки для дальнейшего анализа.
- Поддержка инициатив чаптера по улучшению качества аналитической документации.

### Примеры анализируемых параметров

- Количество страниц, созданных или обновлённых аналитиком.
- Распределение материалов по Confluence space / проектной области.
- Наличие обязательных или рекомендованных макросов.
- Использование согласованных документационных паттернов.
- Страницы, пригодные как референсные примеры.
- Страницы, требующие структурной доработки или очистки.

### Технический подход

- Интеграция с Confluence REST API.
- Аутентификация через корпоративный механизм доступа.
- Фильтрация материалов по автору / доменному логину.
- Парсинг Confluence storage-format content.
- Экспорт результатов в структурированный формат для последующего анализа.

### Ограничения и этическая рамка

Утилита предназначалась для анализа документационных практик и улучшения стандартов чаптера, а не для автоматического скоринга сотрудников или скрытого контроля.

### Что показывает проект

Проект показывает практический подход к documentation governance: лёгкая автоматизация позволяет сделать документационные практики наблюдаемыми, измеримыми и улучшаемыми.

Он также отражает мой рабочий стиль: если процессная проблема повторяется, я предпочитаю формализовать её, извлечь данные и собрать небольшой инструмент, который переводит субъективное обсуждение в evidence-based улучшение.
