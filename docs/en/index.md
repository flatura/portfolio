# Dmitry M. — System Analyst / Solution Design Analyst

I work on system analysis and solution design for enterprise and product systems.

My strongest area is turning complex business domains into implementable system models: data structures, roles, user scenarios, integration flows, API contracts, constraints, edge cases, documentation, and development-ready tasks.

I have a hands-on engineering background: years of infrastructure and internal IT systems experience, Java/Spring, SQL, PostgreSQL, integration specifications, GIS/data tooling, and applied prototypes. This allows me to work not only with requirements, but also with the technical consequences of design decisions.

## Professional Focus

* System analysis for enterprise and B2B systems
* Technical solution design
* Data models, reference data, and entity lifecycles
* API contracts and integration flows
* RBAC, access rights, and context-specific constraints
* Data import, validation, processing, and traceability
* Development-oriented documentation: C4, DFD, BPMN, ADR, specifications
* Backend-aware analysis: design with implementation, operations, and support in mind

## Where I Bring the Most Value

I am useful in situations where simply “collecting requirements” is not enough.

Typical cases:

* the domain is complex, ambiguous, or poorly formalized;
* a business idea has to be translated into a system structure;
* processes, data, roles, integrations, and constraints have to be connected;
* development teams need clear models, contracts, and scenarios;
* an existing solution has to be analyzed, stabilized, or prepared for further development;
* there is a missing technical layer between business, analysis, architecture, and engineering.
